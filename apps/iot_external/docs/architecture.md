# Архитектура проекта

## Цель

Сайт нужен как демонстрационная панель для курсового проекта по управлению микроклиматом помещения с применением RL.

## Слои

### `frontend/`

Назначение:
- показывать текущие значения температуры, влажности и CO2;
- отображать состояние исполнительных устройств;
- переключать режимы `auto/manual`;
- отправлять команды ручного управления;
- объяснять логику RL в наглядной форме.

На этапе 1:
- создан каркас `Vite + React + TypeScript`;
- заложены директории под компоненты, типы, API и мок-данные.

### `backend/`

Назначение:
- отдавать API для демонстрационного фронтенда;
- принимать телеметрию от симулятора или ESP32;
- хранить текущее состояние системы;
- позже выступать точкой интеграции для RL-модуля и истории данных.

На этапе 1:
- создан Flask-каркас;
- добавлен `app factory`;
- добавлены маршруты `health` и `demo snapshot`.

### `simulator/`

Назначение:
- генерировать правдоподобные данные о микроклимате до готовности аппаратной части;
- эмулировать изменение температуры, влажности, CO2 и состояния реле;
- позже уметь публиковать данные в API или брокер.

## Поток данных

На текущем этапе логически предполагается такой поток:

1. `ESP32` или `simulator` формирует JSON-телеметрию.
2. Данные попадают в `backend` или в MQTT-брокер.
3. `frontend` получает актуальное состояние через API или WebSocket.
4. В ручном режиме пользователь отправляет команду управления.
5. Команда уходит в управляющий канал для устройства.

## Интеграция с MQTT/WebSocket

С учетом задачи из курсовой фронтенд будет готовиться к подключению к локальному брокеру по WebSocket.

На текущий момент с ML-модулем уже согласованы:
- сенсоры: `iot_proj/sensors`
- действия: `iot_proj/actions`

Если сайту потребуется более богатая внутренняя модель данных, это лучше делать через адаптер на `backend`, а не ломать контракт RL-модуля.

## Визуальное направление

Каркас будет опираться на `DESIGN.md`:
- крупная типографика;
- светлые полотна и черные/графитовые витринные секции;
- минимум декоративных эффектов;
- один акцентный синий цвет для действий и интерактивных элементов.
